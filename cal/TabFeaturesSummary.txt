캘린더 탭
---------
- 월간 달력에서 이전/다음, 오늘 이동, 연도 선택 시트를 지원하며 하루별 일정 개수를 표시합니다 (`CalendarMonthView`).
- 특정 날짜를 고르면 하루 요약 화면(`DayDetailView`)이 열려 할 일과 Canvas 과제를 분리해서 보여주고, 새 할 일 추가 및 시작·일시정지·다시시작·완료 버튼으로 시간 추적을 제어할 수 있습니다.
- “Today’s countdown” 카드가 오늘 마감되는 할 일과 과제를 모아 남은 시간을 게이지로 표시하고, 완료 토글·상세 화면 이동과 함께 스톱워치 조작을 제공합니다.
- 활성 타이머를 가진 항목은 “Started” 섹션에 모아 동일한 제어를 지원하며, 지연/완료 상태를 강조합니다.
- 행을 탭하면 `TodoDetailView`·`AssignmentDetailView`로 이동해 세부 편집을 수행하고, 스와이프/컨텍스트 메뉴로 완료 토글을 빠르게 처리할 수 있습니다.

할 일 탭
--------
- 하나의 탭 안에서 개인 할 일과 학교 과제를 세그먼트로 전환하며 작업합니다 (`CombinedTasksView`).
- 개인 할 일 목록(`TodoListView`)은 제목, 마감, 우선순위, 추정 시작/종료, 이동 시간, 지연 여부 등을 보여주고, 체크 버튼으로 완료 상태를 토글합니다.
- 스와이프 삭제/편집, 컨텍스트 메뉴를 제공하며, 추가·편집 시트(`TaskEditorSheet`)에서 시작/마감 제공 여부, 우선순위, 예상 소요, 장소, 이동 시간을 설정할 수 있습니다.
- 장소 검색, 위치 권한 요청, 장소 제거, 지도 기반 주행/도보 이동 시간 재계산을 지원하여 실사용 시나리오를 커버합니다.
- 학교 과제(`SchoolWorkView`)는 Canvas ICS 파일/URL을 통해 가져오고, 마지막 동기화 정보를 저장해 한 번의 탭으로 재동기화하며, 네트워크 실패 시 저장된 ICS로 대체합니다.
- 과제는 날짜별로 그룹화되어 종일/시간제 설명, 예상 소요, 과목/장소/설명 요약, 지연·완료 배지를 표시하고, 스와이프 제스처로 완료를 토글합니다.
- `AssignmentDetailView`에서 완료 처리, 예상 소요 시간 선택, Canvas 링크 열기, 전체 설명 확인이 가능합니다.

루틴 탭
--------
- 1·3·7일 보기 전환 가능한 주간 그리드가 루틴을 시각화하며, 현재 시각 라인과 날짜 탭으로 빠르게 이동합니다 (`RoutineView`, `WeeklyScheduleGrid`).
- 상단에는 단일 일 보기 종료 버튼, 보기 전환 메뉴, 플로팅 추가 버튼이 있어 즉시 루틴을 만들 수 있습니다.
- 루틴 폼은 제목, 시작/종료 시간, 반복 요일, 색상, 아이콘을 설정하고 저장하며, 기존 루틴 편집 시 하단에 삭제 버튼이 노출됩니다.
- 그리드 블록은 자정 넘김 루틴을 분할 표시하고, 비활성 항목은 투명도를 낮춰 구분하며, 탭 시 해당 루틴 편집 시트가 열립니다.

챗 탭
------
- 대화 UI(`ScheduleChatView`)가 사용자/어시스턴트 말풍선을 시간 순으로 표시하고 자동 스크롤하며, 저장된 OpenAI API 키를 사용합니다.
- API 키가 없을 때는 간단한 명령 세트로 오늘 할 일 조회, 추가, 완료 처리 등을 로컬 데이터로 즉시 대응합니다.
- API 키가 있을 때 `ScheduleAssistant`가 구조화된 시스템 프롬프트로 요청을 만들고, `<schedule_actions>` JSON을 파싱해 todo.add, todo.toggle, assignment.setDuration 같은 작업을 자동 실행하며 요약 메시지를 첨부합니다.
- 시트를 통해 API 키를 등록·변경할 수 있고, OpenAI 요청 중 로딩 상태와 오류 메시지를 명확히 안내합니다.

설정 탭
--------
- 앱 전체 색상 모드를 시스템 연동/라이트/다크 중에서 선택해 `AppStorage`를 통해 전체 UI에 반영합니다.
- 캘린더 날짜 셀의 표시 방식(원형 중앙, 기호형 중앙, 하단 점)을 지정하여 월간 달력의 이벤트 표시 스타일을 제어합니다.
- 각 항목마다 보조 설명을 제공하여 기술 지식이 없는 사용자도 설정 의도를 이해하기 쉽게 했습니다.
